# Проверка module_2

Скрипт запускается на HQ-CLI и проверяет задания с 1 по 13. Пункт 14 выводится как `N/A`.

Проверьте адреса и учётные данные в `.env`, затем:

```bash
chmod +x ./01-check-module-2.sh
su -
cd /путь/module_2_test
./01-check-module-2.sh
```

При отсутствии диагностических утилит скрипт установит `openssh-clients`, `sshpass`, `expect`, `curl` и `bind-utils`.

Проверяемые службы не изменяются. Временный Expect-файл удаляется после завершения.

Проверка reverse proxy поддерживает оба состояния стенда:

- после module_2 проверяется HTTP;
- после module_3 автоматически проверяется редирект на HTTPS и доверие сертификату без отключения TLS-проверки.

Если прямой SSH к серверу недоступен, скрипт автоматически пробует статический NAT маршрутизатора. Адреса можно явно задать через `HQ_SSH_HOST`, `HQ_SSH_PORT`, `BR_SSH_HOST` и `BR_SSH_PORT` в `.env`. SSH на ISP необязателен; включите `ISP_SSH_ENABLED=yes` только если он настроен.

Проверить только формат итоговой таблицы без сетевых подключений:

```bash
./01-check-module-2.sh --self-test
```
